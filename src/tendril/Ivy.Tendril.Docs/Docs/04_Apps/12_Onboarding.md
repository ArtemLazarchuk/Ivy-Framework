# Onboarding

<Ingress>
First-run wizard when `TENDRIL_HOME` is missing or invalid.
</Ingress>

## Steps

1. **`TENDRIL_HOME`** — Where Plans, config, costs, and Trash live.
2. **API keys** — e.g. Anthropic for promptware.
3. **First project** — Repo to track.

Then Tendril writes `config.yaml` and continues.

## Skip

If `TENDRIL_HOME` already has a valid `config.yaml` (e.g. from dotfiles or IT), onboarding is skipped and you go straight to the Dashboard.
